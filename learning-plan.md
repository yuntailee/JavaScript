# 前端學習計畫（表單／互動／jQuery 導向）

**時間假設**

- 平日：每天 **1 小時**（建議週一至週五，若某天請假就順延，不必自責）。
- 週末：每週約 **3 小時**（可全放在週日，或拆成 1.5 + 1.5；週六晚則以休息或 30 分鐘複習為主即可）。

**每週總量約 8 小時**：5×1 + 3。

**按日期資料夾**：`daily-schedule/` 內依日期建立子資料夾，每日內容與作業見該資料夾的 `README.md`。

**使用方式**

- 每天只做「當天主題」，做完就停，避免過量。
- 所有練習請用**自己電腦上的新檔案**，不要複製公司程式碼。
- 卡關時把**自己寫的簡化版**貼到 Cursor 問即可。

## 第 1 週：JavaScript 核心 + DOM 入門

| 時段 | 長度 | 內容 |
|------|------|------|
| 週一 | 1h | 變數、`let`/`const`、型別、字串與樣板字串；在 Console 試算。 |
| 週二 | 1h | 函式、`if/else`、陣列 `map/filter` 擇一練熟。 |
| 週三 | 1h | 物件、存取屬性；練一個「表單欄位名 → 值」的小物件。 |
| 週四 | 1h | **DOM**：`querySelector` / `querySelectorAll`、改 `textContent`、加 `classList`。 |
| 週五 | 1h | **事件**：`addEventListener`（`click`）、`submit` 表單、`preventDefault`。 |
| 週末 | 3h | **小專題**：一個 HTML 頁面，兩個輸入框 + 一個按鈕，按下去在畫面上顯示「你好，{名字}」；順便整理本週筆記。 |

**週末 3 小時建議切法**：50 分鐘寫作 + 10 分鐘休息 × 3 輪，最後 30 分鐘複習 MDN 上抄下的重點。

---

## 第 2 週：表單驗證（原生 JS）

| 時段 | 長度 | 內容 |
|------|------|------|
| 週一 | 1h | 必填檢查：空字串、去掉前後空白 `trim()`。 |
| 週二 | 1h | 在欄位旁或下方顯示錯誤訊息（用 DOM 插入或切換 class）。 |
| 週三 | 1h | **Regex 入門**：只允許數字、或簡單 Email 格式（二選一練到熟）。 |
| 週四 | 1h | `input` 事件即時驗證 vs `submit` 一次驗證，兩種都試。 |
| 週五 | 1h | 送出時若錯誤則挡下來、全對才 `console.log` 模擬送出。 |
| 週末 | 3h | **小專題**：註冊表單（姓名、手機、Email、密碼），每項規則不同，錯誤要清楚；程式分小函式（例如 `validatePhone`）。 |

---

## 第 3 週：jQuery（對齊你工作情境）

| 時段 | 長度 | 內容 |
|------|------|------|
| 週一 | 1h | 在頁面引入 jQuery（CDN）；`$` 選取、`val()`、`text()`。 |
| 週二 | 1h | `.on('click')`、`.on('submit')`；與原生寫法對照寫一次。 |
| 週三 | 1h | 表單：`.serialize()` 或逐欄位取值，印到 Console。 |
| 週四 | 1h | 若教材有：`$.ajax` 或 `$.post` 讀官方範例（可先對假 API 或 JSONPlaceholder）。 |
| 週五 | 1h | 把「第 2 週註冊表單」**改寫成 jQuery 版**（邏輯可相同）。 |
| 週末 | 3h | **小專題**：登入表單（帳密）+ 前端驗證 + 一個「假送出」按鈕（`$.post` 到測試 API 或僅 success callback 裡 `alert`）；整理 jQuery 與原生差異筆記。 |

---

## 第 4 週：整合 + 往全端鋪路

| 時段 | 長度 | 內容 |
|------|------|------|
| 週一 | 1h | 複習：事件冒泡、`stopPropagation` 何時用（查 MDN）。 |
| 週二 | 1h | **JSON**：`JSON.parse` / `JSON.stringify`；假裝後端回傳一筆使用者物件。 |
| 週三 | 1h | **HTTP 粗概念**：GET/POST、狀態碼 200/400/500 意義（影片或 MDN 短文即可）。 |
| 週四 | 1h | `fetch` 發 GET 到公開 API，把結果顯示在頁面上（可先不用 jQuery）。 |
| 週五 | 1h | 整理「前端驗證 vs 後端還要驗」清單，寫成給自己看的五條 bullet。 |
| 週末 | 3h | **小專題**：一頁「查天氣／查匯率／假資料列表」任一（用 `fetch`），加簡單載入中與錯誤提示；回顧四週，標註下月想加強的一點（例如 Regex 或 async）。 |
